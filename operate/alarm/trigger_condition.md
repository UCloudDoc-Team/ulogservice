# 触发条件表达式

触发条件用于定义告警触发规则，当执行语句的查询结果满足触发条件时，系统将发送告警通知。

## 语法规则

触发条件表达式支持使用比较运算符和逻辑运算符组合判断条件。

### 比较运算符

| 运算符 | 说明 | 示例 |
| --- | --- | --- |
| `==` | 等于 | `count == 10` |
| `!=` | 不等于 | `status != 200` |
| `>` | 大于 | `count > 100` |
| `>=` | 大于等于 | `count >= 100` |
| `<` | 小于 | `latency < 1000` |
| `<=` | 小于等于 | `latency <= 1000` |

### 算术运算符

| 运算符 | 说明 | 示例 |
| --- | --- | --- |
| `+` | 加法 | `count + 10` |
| `-` | 减法 | `total - error` |
| `*` | 乘法 | `count * 2` |
| `/` | 除法 | `total / 10` |
| `%` | 取模 | `count % 100` |

### 逻辑运算符

| 运算符 | 说明 | 示例 |
| --- | --- | --- |
| `&&` | 逻辑与，两个条件都满足时触发 | `count > 100 && status == 500` |
| `\|\|` | 逻辑或，满足任一条件即触发 | `status == 500 \|\| status == 502` |

### 其他运算符

| 运算符 | 说明 | 示例 |
| --- | --- | --- |
| `()` | 括号，用于改变运算优先级 | `(count + error) > 100` |

## 内置字段

### __QUERYCOUNT__

`__QUERYCOUNT__` 表示执行语句查询返回的日志条数。

**使用示例：**

| 触发条件 | 说明 |
| --- | --- |
| `__QUERYCOUNT__ == 1` | 查询结果正好返回 1 条日志时触发告警 |
| `__QUERYCOUNT__ > 10` | 查询结果返回超过 10 条日志时触发告警 |
| `__QUERYCOUNT__ >= 5` | 查询结果返回 5 条及以上日志时触发告警 |

**注意事项：**

- 不支持 `__QUERYCOUNT__ == 0` 的用法，即无法通过"无查询结果"触发告警
- `__QUERYCOUNT__` 仅用于判断日志条数，不能与其他字段混用

## 使用示例

### 示例一：错误日志监控

执行语句：
```
status >= 400 | select count(*) as error_count
```

触发条件：
```
error_count > 100
```

说明：当错误日志数量超过 100 条时触发告警。

### 示例二：响应时间监控

执行语句：
```
* | select avg(latency) as avg_latency
```

触发条件：
```
avg_latency > 1000
```

说明：当平均响应时间超过 1000ms 时触发告警。

### 示例三：特定日志出现告警

执行语句：
```
level: ERROR
```

触发条件：
```
__QUERYCOUNT__ >= 1
```

说明：当出现 ERROR 级别日志时触发告警。

### 示例四：多条件组合

执行语句：
```
* | select count(*) as total, sum(case when status >= 500 then 1 else 0 end) as server_error
```

触发条件：
```
total > 1000 && server_error > 10
```

说明：当日志总数超过 1000 条且服务端错误超过 10 条时触发告警。
