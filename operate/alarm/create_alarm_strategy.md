# 创建告警策略

本文档介绍如何创建告警策略，配置日志监控规则和告警通知方式。

## 前提条件

- 已创建日志集和日志主题
- 日志主题已有数据接入

## 操作步骤

1. 登录日志服务控制台。
2. 在顶部TAB中，选择 **日志告警告警**。
![image](/images/alarm/alarm_1.png)
3. 左侧TAB中选择告警策略。
![image](/images/alarm/alarm_2.png)
4. 点击 **创建告警策略** 按钮。
![image](/images/alarm/alarm_3.png)
5. 填写告警策略的基本信息。

| 配置项 | 说明 |
| --- | --- |
| 策略名称 | 设置告警策略的名称，支持中文、英文、数字和下划线 |
| 日志主题 | 选择需要监控的日志主题 |

6. 填写告警配置。

| 配置项 | 说明 |
| --- | --- |
| 是否启用 | 选择是否启用该告警策略，启用后策略将按配置执行监控 |
| 执行语句 | 输入用于检索分析的 SQL 语句，语法规则与检索分析一致。一个告警策略最多只能添加3条执行语句。<br />1. 单机添加执行语句。<br />2. 填写执行语句，语法可参考：[检索分析](/ulogservice/operate/search)。<br />3.选择时间范围，最大时间范围是1天。<br />4. 选择预览主题对象，点击预览。<br />5. 点击确定添加一条执行语句。<br /> |
| 触发条件 | 设置告警触发的条件表达式，当查询结果满足条件时触发告警。<br /> **注意:** <Br />如果您想要有特定条数据触发告警，请使用使用__QUERYCOUNT__字段来判断，比如条数等于1，使用__QUERYCOUNT__==1语句，不支持__QUERYCOUNT__==0的用法。<br /> 具体语法参考：[触发条件表达式](/ulogservice/operate/alarm/trigger_condition)|
| 执行周期 | 设置策略执行的时间间隔，如每 1 分钟、每 5 分钟等 |

1. 填写告警通知。

| 配置项 | 说明 |
| --- | --- |
| 告警频率 | 设置告警通知的频率，如持续1个监控周期满足触发条件，则每15分钟告警一次 |
| 附加通知内容 | 可选，告警时附加的内容，支持填写变量，可使用变量参考：[附加通知内容变量](/ulogservice/operate/alarm/notification_variable) |

8. 单机确定按钮完成告警策略创建。
创建告警策略后，如果告警策略时启用状态，日志服务会根据告警策略持续运行监控任务并发送告警通知。
