# 附加通知内容变量

在告警策略配置中，您可以在附加通知内容中使用变量，告警通知发送时会将变量替换为实际的值。

## 可用变量列表

| 变量 | 类型 | 说明 |
| --- | --- | --- |
| `{{Alarm}}` | string | 告警策略名称 |
| `{{AlarmID}}` | string | 告警策略 ID |
| `{{Level}}` | string | 告警级别 |
| `{{ExecuteQuery}}` | []string | 执行语句列表 |
| `{{QueryResultStr}}` | []string | 查询结果列表 |
| `{{Condition}}` | string | 触发条件表达式 |
| `{{StartTime}}` | string | 告警开始时间 |
| `{{Topic}}` | string | 日志主题 |
| `{{Duration}}` | string | 告警持续时间 |
| `{{NotifyTime}}` | string | 通知发送时间 |

## 使用方式

在附加通知内容中直接输入变量名称即可，例如：

```
告警策略：{{Alarm}}
告警级别：{{Level}}
触发时间：{{StartTime}}
日志主题：{{Topic}}
触发条件：{{Condition}}
```

## 通知示例

假设您配置了以下附加通知内容：

```
【告警通知】
...
附加通知内容：
策略名称：{{Alarm}}
告警级别：{{Level}}
触发时间：{{StartTime}}
日志主题：{{Topic}}
触发条件：{{Condition}}
```

实际收到的告警通知将显示为：

```
...
附加通知内容：
策略名称：错误日志监控
告警级别：紧急
触发时间：2026-05-14 10:30:00
日志主题：uls-1o4h9cdkn8jk
触发条件：$1.error_count > 100
请及时处理！
```

## 注意事项

- 变量名称区分大小写，请确保输入正确
- 建议在配置完成后进行告警测试，确认通知内容格式正确
