# 查看告警历史

本文档介绍如何查看告警历史记录，追溯问题发生的时间和详情。

## 功能说明

告警历史记录了所有触发告警的详细信息，包括告警时间、告警内容、触发条件等，帮助用户分析和定位问题。

## 操作步骤

1. 登录日志服务控制台。
2. 在顶部TAB中，选择 **日志告警告警**。
![image](/images/alarm/alarm_1.png)
3. 左侧TAB中选择告警记录。
![image](/images/alarm/alarm_6.png)

### 告警历史列表

告警历史列表展示以下信息：

| 字段 | 说明 |
| --- | --- |
| 发生时间 | 告警的发生时间。 |
| 告警策略 | 触发告警的策略名称 |
| 监控对象 | 被监控的日志主题或资源 |
| 触发条件 | 触发告警的规则条件 |
| 告警频率 | 告警通知的发送频率 |
| 持续时间 | 告警持续的时间长度 |
| 级别 | 告警严重程度，如紧急、重要、一般 |
| 状态 | 告警的处理状态 |
| 操作 | 可以查看执行详情。一条告警由告警策略+主题ID唯一标识，相同的告警策略+主题ID的告警记录会归属到同一个告警下，点击执行详情，可以看到该告警下的所有执行记录。可以查看的信息如下：<br /> 1. 执行时间：告警策略执行语句的执行时间。<br /> 2. 监控对象：主题ID。<br />3. 触发条件：具体的触发条件语句。<br /> 4. 是否触发告警：当前这一次执行是否满足触发条件，如果满足，字段为是，不满足为否。 |

### 筛选查询

支持通过以下条件筛选告警历史：

- **时间范围**：选择特定时间段的告警记录
- **策略名称**：按告警策略筛选
- **日志主题**：按日志主题筛选
- **级别**: 按照告警级别筛选
- **状态**：按告警状态筛选

### 查看详情

点击告警记录，可查看告警详情，包括：

- 告警触发的查询语句和结果
- 触发条件的具体值
- 通知发送状态
- 相关日志内容

## 注意事项

- 告警记录历史只保留 30 天
