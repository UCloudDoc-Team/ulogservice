# 修改告警策略

本文档介绍如何修改已创建的告警策略，包括修改监控规则、触发条件和通知配置等。

## 操作步骤

1. 登录日志服务控制台。
2. 在顶部TAB中，选择 **日志告警告警**。
![image](/images/alarm/alarm_1.png)
3. 左侧TAB中选择告警策略。
![image](/images/alarm/alarm_2.png)
4. 在告警策略列表中，找到需要修改的策略，点击 **详情** 按钮。
![image](/images/alarm/alarm_4.png)
5. 在告警策略详情里可以对指定字段进行修改：
![iamge](/images/alarm/alarm_5.png)

### 基本信息

| 配置项 | 说明 |
| --- | --- |
| 策略名称 | 修改告警策略的名称，支持中文、英文、数字和下划线 |
| 日志主题 | 修改需要监控的日志主题 |

### 告警配置

| 配置项 | 说明 |
| --- | --- |
| 是否启用 | 修改是否启用该告警策略 |
| 执行语句 | 修改用于检索分析的 SQL 语句 |
| 触发条件 | 修改告警触发的条件表达式 |
| 执行周期 | 修改策略执行的时间间隔 |

### 告警通知

| 配置项 | 说明 |
| --- | --- |
| 告警频率 | 修改告警通知的频率 |
| 附加通知内容 | 修改告警时附加的内容 |

6. 修改完成后，点击 **确定** 保存修改。

## 注意事项

- 修改告警策略后，新的配置将在下次执行时生效
- 修改过程中不会影响当前正在执行的告警任务
- 建议在修改后验证告警策略的正确性
